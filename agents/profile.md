# Mike Pawliszyn -- Developer Profile

Staff engineer, ~18 years experience (Amazon ~8, Block/Cash App ~10).

This profile is tool-agnostic. It ships to Claude Code via symlink
to ~/.claude/CLAUDE.md and can be routed to other AI tools as needed.

## Growth Orientation

I want to learn. I expect agents to learn too -- through feedback loops,
session retros, and updated documentation. Growth is bidirectional.

## Working Style

- **Ask, don't assume.** If something is ambiguous, ask. I'd rather
  answer a question than undo a wrong assumption.
- **Customer, not user.** The vocabulary keeps the right mindset.
- **Clean PRs.** One logical change per commit. Descriptive messages.
- **Explain your reasoning.** Show your work. I want to understand
  every change.

## Values

- **Truth.** Core value. Evidence over anecdote.
- **Rigour.** Evidence-based decisions. Measure twice.
- **Conciseness.** Pick the right word. Say it once.
- **Whimsy.** Fun names that guide understanding.
- **Usability.** Prevention over blame. Guardrails over lectures.

## Development Philosophy

- **YAGNI.** Build what's needed now.
- **Fakes over mocks.** Strong testing opinion.
- **Sandboxed integration tests.** Test what customers need.
- **Tool-agnostic.** Don't couple to any single tool or vendor.
- **Docs hierarchy.** README is the foundation. AGENTS.md layers on top.

## Session Retro

At the end of sessions, reflect: do docs, agent files, or memory
need updates? Report reasoning, not just actions.
